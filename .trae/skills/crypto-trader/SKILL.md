# AI 加密货币合约交易系统

## 项目概述

一个全自动化的 Binance U本位合约量化交易系统，通过 AI（DeepSeek）分析币安广场社交动态 + 24h涨跌榜 + 历史K线波动数据，自动判断市场情绪并执行交易。

**核心逻辑**：采集币安广场动态 + 涨跌榜Top5 → K线阶段分析 → AI分析情绪 → 冷却双确认 → 自动交易（20%余额开仓，5x杠杆）→ 3秒动态风控（止损30%/止盈40%/追踪50%/回撤15%）→ 1h/3h/6h/12h/24h微信汇总（含AI复盘）

## 风险管理策略

以 **20 USDT 保证金（余额 100u，开仓20%），5x 杠杆** 为例：

| 条件 | 操作 | 示例 |
|------|------|------|
| **开仓** | 使用 20% 可用余额作为保证金 | 余额 100u → 开仓保证金 20u |
| **止损** | 亏损 ≥ 30% 当前保证金 = 6% 总余额 | 亏 6u → 自动平仓，进入冷却状态 |
| **止盈减仓** | 盈利 ≥ 40% 初始保证金 = 8% 总余额 | 赚 8u → 减仓 55% 保证金（减 11u，余 9u），止损移至开仓价（保本），退出冷却 |
| **追踪启动** | 剩余仓位盈利 ≥ 50% 当前剩余保证金 | 余 9u，浮盈 ≥ 4.5u 时激活追踪 |
| **追踪平仓** | 浮盈从最高点回撤 ≥ 15% | 最高浮盈 8u，回撤至 6.8u 时 → 全部平仓，退出冷却 |
| **服务器止损** | 开仓后自动设 STOP_MARKET（30%保证金），程序崩溃也生效 | 主网支持，测试网降级 |

## 冷却双确认机制

止损后不等待时间，而是要求下一单必须满足**双重确认信号**才允许开仓，盈利一单后自动恢复：

- **A（必须）**: 价格站上MA5 + 收盘突破前高
- **B 或 C（至少一个）**: RSI>35+MACD金叉 / 放量突破

## 4个并行循环

| 循环 | 频率 | 职责 |
|------|------|------|
| **AI分析循环** | 每15分钟 | 涨跌榜Top5 + 广场动态 → AI分析情绪+开新仓；空仓时立即建仓 |
| **市场分析循环** | 每3分钟 | 获取市场数据 + 历史波动 → AI判断是否强制止盈减仓 |
| **风险监控循环** | 每3秒 | 止损(30%)/止盈(40%减仓55%)/追踪(50%激活/15%回撤) |
| **汇总报表循环** | 每分钟检查 | 到点发 1h/3h/6h/12h/24h 微信汇总（含AI复盘分析） |

## 运行方式

### 前提条件

1. 安装 Python 依赖：`uv sync` 或 `pip install -r requirements.txt`
2. （自动）Chrome 调试模式由系统自动启动（端口 9555）
3. 币安网站已登录（广场关注页有数据）

### 命令

```bash
# 方式1：无限循环（4个并行循环，生产模式）
uv run python scheduler.py forever

# 方式2：手动执行一轮完整流程
uv run python scheduler.py

# 方式3：查看交易统计
uv run python -c "from utils.trade_records import print_summary; print_summary()"

# 方式4：查余额
uv run python check_balance.py
```

### 单轮流程

每轮 AI 分析循环自动执行：
1. **涨跌榜** — 获取24h涨跌幅榜Top5，分析K线阶段
2. **采集** — 从币安广场获取最新动态（如有）
3. **AI 分析** — DeepSeek 分析涨跌榜+动态+历史波动，输出情绪和交易建议
4. **冷却检查** — 冷却中需双确认信号才开仓
5. **信号确认** — 等待120秒确认价格未反向波动超过3%
6. **K线入场分析** — 检查K线趋势突破（MA6/MA20），横盘则等180秒重查或换动态候选币
7. **交易执行** — 开多/开空/持有（持仓中禁止开新单）
8. **服务器止损** — 开仓后设30% STOP_MARKET 止损单

## 配置说明

`config.py` 包含所有敏感信息：

- `TESTNET_API_KEY / TESTNET_SECRET` — 测试网密钥
- `MAINNET_API_KEY / MAINNET_SECRET` — 实盘密钥（⚠️ 注意保护）
- `DEEPSEEK_API_KEY` — AI 分析的 API Key
- `PUSHPLUS_TOKEN` — 微信通知 Token
- `USE_TESTNET` — `True` 测试网 / `False` 实盘

## 调用方式

作为 AI，当用户提出以下需求时调用此 Skill：

1. **"运行交易系统"** → 执行 `uv run python scheduler.py forever`
2. **"查当前持仓/盈亏"** → 执行查持仓 + 读 `data/risk_state.json`
3. **"查看交易统计/胜率"** → 执行 `from utils.trade_records import print_summary; print_summary()`
4. **"调整策略参数"** → 修改 `auto_trader.py` 或 `risk_monitor.py` 中的参数
5. **"切换实盘/测试网"** → 修改 `config.py` 中的 `USE_TESTNET`
6. **"查分析结果"** → 读取 `data/trade_signals.json` 最新信号
7. **"查风险状态"** → 读取 `data/risk_state.json`
8. **"发汇总报表"** → 查看 `data/analysis_summary.jsonl` 最新复盘分析
