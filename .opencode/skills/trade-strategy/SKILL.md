# 交易策略层 — 执行与风险管理

根据 AI 信号执行开平仓，内置完整的风险管理体系：三阶段止盈、信号确认延迟、方向反转保护、冷却双确认、K线入场分析、服务器端止损。

## 适用场景

- 用户要求"现在策略是怎么跑的"、"止盈止损怎么设的"
- 用户要求"调整一下止损比例"、"把止盈改小点"
- 用户要求"看看当前风险状态"、"检查持仓风控"
- 用户要求"快捞参数是多少"、"快捞的止盈止损比例"
- 用户要求"冷却时间到了吗"、"双确认过了没"
- 系统自动调用：风险监控每1.5秒、快捞每2分钟

## 安全约定

- 持仓中禁止开新单（由 auto_trader 强制执行）
- 止损后进入冷却（`is_cooling()`），需双确认信号才允许开仓
- 服务器端止损（STOP_MARKET）开仓后自动设置
- AI 强制减仓需要 `confidence >= 75` 且 `force_close` 为 true

## 输入输出

输入：AI 分析信号（`{action, symbol, confidence}`）

输出：
- 开仓结果：`{executed_action, symbol, quantity, price, order_id, fills}`
- 风险状态：`{symbol, margin, pnl, tp_done, trailing_active, cooling}`
- 风控动作：stop_loss / take_profit / trailing_stop / reduce_position

## 执行流程

### 开仓执行流程（仅快捞）

1. **快捞触发**：15分钟涨跌幅>6% + K线趋势过滤（做多仅up，做空仅down）
2. **动态候选币种**：从 24h 涨跌榜 + 广场热门 + X趋势 实时获取
3. **计算仓位**：开仓保证金 = 5% 可用余额，5x杠杆
4. **执行开仓**：调用 core-trader 的 place_market_order
5. **止损**：-1.5% 始终有效（高频检测）
6. **浮动锁仓**：+10%启动（锁+2%），之后每+5%再+2%

### 风险监控流程（1.5秒循环）

1. **初始小止损**：亏损 ≥ 15% 保证金 → 全平，进入冷却
2. **TP1 首轮止盈**：浮盈 ≥ 45% 保证金（价格正向 ~9%，即止损距离×3，1:3盈亏比）→ 平 50%，止损移至保本
3. **保本止损**：价格回到开仓价 → 全平（盈亏比保底 1:1.5）
4. **防耗散离场**：3根1h K线无方向 / 盘整区 → 全平，不给浪费时间
5. **趋势反转**：K线趋势与持仓方向相反 → 全平
6. **趋势延续**：以上均不触发 → 持有吃大行情

## 止盈策略

| 阶段 | 触发条件 | 操作 |
|------|---------|------|
| **初始止损** | 亏损 ≥ 15% 保证金（约价格反向3%） | 全平，进入冷却 |
| **TP1 首轮止盈** | 浮盈 ≥ 45% 保证金（约价格正向9%，1:3盈亏比） | 平50%，止损移至保本（开仓价） |
| **保本止损** | 价格回到开仓价 | 全平（盈亏比保底 1:1.5） |
| **防耗散** | 3根1hK线无方向 / 盘整区 / 趋势反转 | 全平离场 |
| **趋势延续** | 方向正确，无以上条件 | 持有吃大行情 |

### 策略说明

- **1:3 盈亏比**：止损 = -15%保证金（~3%价格），止盈 = +45%保证金（~9%价格）
- TP1 平半仓后，止损立即移到成本价，0风险博后续行情
- 如果回头打保本，已落袋的 TP1 利润确保盈亏比 ≥ 1:1.5
- 方向正确则一直持有，直到防耗散/趋势反转信号出现才离场

## 快捞系统

独立于主仓的快速小仓位交易，与主仓互不影响。追涨（6%做多）、跟跌（6%做空）情绪币。
采用浮动利润锁仓策略：不做减仓，让利润奔跑，逐步提升保底线。

| 参数 | 值 | 说明 |
|------|-----|------|
| 监测频率 | 每2分钟 | 扫描500+USDT币种 |
| 做多触发 | 15min涨幅>6% + K线趋势up | 对比15分钟前快照 |
| 做空触发 | 15min跌幅>6% + K线趋势down | 对比15分钟前快照 |
| 仓位 | 5%余额，5x杠杆 | 远小于主仓20% |
| 做多/空止损 | -1.5% | 始终有效 |
| **起始锁仓** | **盈利+10%** | **锁仓线+2%（回撤到+2%平）** |
| **提档规则** | **每多+5%盈利** | **锁仓线再+2%** |
| 最低成交额 | ≥50万USDT | 排除流动差的币种 |
| **冷却时间** | **止损后15分钟** | **大亏损(>20U)冷却60分钟** |

### 浮动锁仓策略说明

```
盈利+10% ──► 锁仓线+2%（保底2%利润）
盈利+15% ──► 锁仓线+4%（+5%→+2%锁仓）
盈利+20% ──► 锁仓线+6%
盈利+25% ──► 锁仓线+8%
   ⋮            ⋮
一直涨    ──► 一直持有，锁仓线自动上移
回撤到线  ──► 平仓，落袋为安
```

- **不加仓、不减仓** — 固定5%余额+5x杠杆，进去就持有不动
- **止损 -1.5%** — 始终有效，方向错了第一时间跑
- **动态锁仓** — 每多涨5%，锁仓线就上移2%，越涨锁得越死
- **逐步扩大优势** — 比如涨到30%，锁仓线就到10%，回撤空间只有20%，利润锁得死死的

### 风险机制

1. **K线趋势过滤** — 做多仅K线趋势=up，做空仅趋势=down，逆势不开仓
2. **冷却机制** — 止损后同币种15分钟内不再触发快捞
3. **大亏损长冷却** — 单笔止损 > 20 USDT 时，冷却延长至 60 分钟
4. **过期冷却自动清理** — 每次开仓前清理已过期的冷却记录

## 质量标准

- 开仓前必须检查所有前置条件（冷却/持仓/K线趋势过滤）
- 止损 -1.5%（快捞） / 止损 -15%保证金（主仓）始终有效
- 风险监控每1.5秒执行，不延迟
- AI 强制减仓与自动化止盈互不干扰
- 快捞和主仓仓位独立，互不影响
- 浮动锁仓：+10%启动，每+5%盈利锁仓线+2%
