# AI 加密货币合约交易系统

全自动 Binance U本位合约量化交易系统，由 DeepSeek AI 驱动，分析币安广场社交动态 + 24h涨跌榜 + 实时K线数据，自动判断市场情绪并执行交易。包含 5 个并行循环 + 1 个 Web 看板，覆盖分析、交易、风控、监控、报表全流程。

## 适用场景

- 用户要求"启动交易系统"、"运行机器人"
- 用户要求"看看系统在干嘛"、"系统状态"
- 用户要求"了解这个交易系统"、"系统整体概览"
- 用户询问"量化交易"、"自动交易"、"交易机器人"等关键词
- 用户需要管理系统的启动/停止/状态查看

## 安全约定

- 使用币安 U本位合约，5x 杠杆
- 系统自动止损（15%保证金），避免大幅亏损
- 止损后自动进入冷却期，需双确认才允许重新开仓
- 所有 API Key 从 `config.py` 读取，不暴露在任何文件中
- 系统启动时自动执行对账，确保交易记录完整

## 5个并行循环

| 循环 | 频率 | 职责 | 默认持仓时 | 空仓时 |
|------|------|------|-----------|--------|
| **快捞监控循环** | 每2分钟 | 扫描500+币种，15min涨>6%做多/跌>6%做空（AI分析+趋势过滤+冷却）→开新仓 | 跳过 | 运行 |
| **风险监控循环** | 每1.5秒 | 小止损15%/1:3盈亏比止盈(45%保证金)+保本/防耗散/趋势反转 | 每1.5秒 | 每30秒 |
| **汇总报表循环** | 每分钟检查 | 到点发 1h/3h/6h/12h/24h 微信汇总 | 运行 | 运行 |
| **全场监控循环** | 每1小时 | 检测500+币种1h涨幅>50%，微信预警 | 运行 | 运行 |
| **趋势采集循环** | 每15分钟 | 采集广场热门 + X.com 趋势数据 | 运行 | 运行 |

## 调度器架构

```
scheduler/
├── __init__.py      # 入口：run_forever(), run_once() + 常量
├── loops.py         # 6个并行循环线程函数
├── state.py         # 共享状态（锁、计数器、持久化）
└── conditions.py    # 前置条件检查 + 系统状态打印
```

## 启动流程

1. **前置检查**：API密钥配置、Chrome调试模式、网络连接
2. **数据库初始化**：创建 `trading.db` 等数据文件
3. **交易对账**：自动从交易所拉取历史成交记录，补全本地记录
4. **启动6个并行线程**：分别在后台运行
5. **状态打印**：打印系统启动状态和各循环配置
6. **等待 Ctrl+C**：保持运行，各循环按各自频率执行

## 质量标准

- 所有交易决策必须基于 AI 分析结果
- 持仓时禁止开新单（防止重复建仓）
- 止损后必须进入冷却期
- 汇总报表按时发送，不重复不遗漏
- 异常发生时记录日志并打印错误栈
- 启动时自动对账，保证交易记录完整
